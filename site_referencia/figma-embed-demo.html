Figma Embed × CRT: 4 Approaches

Como harmonizar protótipos coloridos com a estética monocromática verde/preto do terminal.

01. RAW IFRAME

CONTRASTE MÁXIMO — sem tratamento

Iframe Figma puro. Fundo claro, cores vivas. Quebra a imersão CRT completamente.

✗ RUIM — o branco do Figma grita contra o fundo preto. Imersão quebrada.

02. CRT PORTAL WINDOW

ISOLADO + OVERLAY — janela de programa rodando no terminal

Iframe com &theme=dark + overlay scanline + vinheta. O protótipo parece uma "aplicação rodando dentro do terminal CRT".

✓ BOM — o contraste vira parte da narrativa. O terminal está exibindo uma aplicação externa.

03. CRT THUMBNAIL → OPEN EXTERNAL

PRÉVIA CONTROLADA — screenshot + link externo

Thumbnail com grade CRT + scanlines + botão "▶ LOAD PROTOTYPE". O protótipo real abre fora do site.

▶ LOAD PROTOTYPE
✓ BOM — controle visual total. Ideal quando o protótipo é bonito, mas não precisa ser interativo no site.

04. GREEN-TINTED FILTER

FORÇA A PALETA — CSS filter sobre o iframe

grayscale(1) sepia(1) hue-rotate(90deg) força tudo pro verde. Mas distorce o trabalho de UI/UX — cores são parte do design.

⚠ RISCO — unifica o visual, mas distorce o trabalho. Só use se o protótipo for só sobre layout, não sobre cor.

→ Híbrido recomendado: PORTAL + THUMBNAIL

Ao clicar, abre um modal fullscreen com o iframe Figma em dark mode, envelopado em bordas CRT e overlay sutil. Assim você mantém o controle visual da página E oferece o protótipo interativo real quando o visitante quer.
